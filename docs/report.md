# Порядок оформления отчета

Отчет оформляется по каждой выполненной практике отдельно и представляет собой текстовый файл в формате markdown, содержащий должным образом оформленный текст и картинки. Требования к содержимому отчета указаны в практике по каждой теме.

Markdown представляет собой легкий для чтения и освоения язык разметки. К примеру, вот [страница, содержащая шпаргалку по этому формату](https://gist.github.com/Jekins/2bf2d0638163f1294637), а также [онлайн-редактор для предварительного просмотра оформленного текста](https://markdownlivepreview.com/).

> Кстати, все материалы в этом репозитории оформлены именно в формате Markdown.

Отчеты по каждой теме в виде файлов с раширением `.md` следует помещать в каталог `/report`, который уже есть в репозитории. Сами файлы отчетов по каждой теме тоже созданы, содержат заголовок темы и готовы для редактирования.

Картинки, которые должны быть вклчены в отчет, следует сохранять в каталог `/report/img` под осмысленным именем, после чего включать в отчет.

Пример оформления отчета по теме приведен в файле `/report/example.md`. Обратите внимание, что две картинки, включенные в этот отчет, сохранены в `/report/img`.

Когда все темы [на требуемую оценку](grades.md) выполнены, все отчеты по этим темам должны быть слиты в единый документ `.docx` или `.pdf` (пояснительную записку). Происходит защита курсовой работы, после чего ПЗ загружается в личный кабинет ГУАП.
