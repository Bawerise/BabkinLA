# 3) Конфигурирование сервера

1. Получите с помощью `psql` список параметров, для изменения которых требуется перезапуск сервера (запросом к `pg_settings` с условием `context='postmaster'`)
2. Проверьте реакцию сервера на ошибку конфигурации
    - в файле `config/postgresql.conf` допустите синтаксическую ошибку при изменении `max_connections`
    - перезапустите сервис
    - проверьте состояние сервиса
    - проверьте его логи
    - исправьте ошибку
    - перезапустите сервис
3. Конфигурирование параметра уровня сеанса
    - установите параметр `work_mem` для текущего сеанса psql
    - проверьте его значение
    - выйдите из psql
    - зайдите в psql повторно
    - проверьте значение параметра `work_mem`
